# Häufig gestellte Fragen, Einschränkungen und Best Practices

Häufig gestellte Fragen zu und Best Practices für Kundenattribute in Analytics und Target.

## Best Practices und Einschränkungen  {#section_7F5189B3DAA84EE6865B91D2026EE05A}

Hinweise und Einschränkungen bei der Verwendung von Kundenattributen.

| Fehler | Beschreibung |
|--- |--- |
| Kundenattributbeschränkungen - Abonnement | Wenn Sie ein Upgrade auf Analytics Premium durchführen, erfolgt eine Verzögerung von 24 Stunden, bevor zusätzliche Attribute verfügbar sind. Während dieser Wartezeit wird Ihnen möglicherweise eine Fehlermeldung über einen erreichten Attributabonnement-Maximalwert angezeigt. |
| Mehrere Anmeldungen auf demselben Gerät | Wenn Sie Kundenattribute zum Hochladen von Profilen in eine Datenquelle verwenden, empfiehlt Adobe, dass Benutzer dasselbe Gerät (d. h. dieselbe Experience Cloud-ID) verwenden. Dies kann dazu führen, dass der auf dem Gerät vorhandene ECID-Dienst mehrere Benutzer unter derselben Experience Cloud-ID verknüpft, was zu unerwarteten Ergebnissen führt [!DNL Target]. **Hinweis:** Bei Mobile ist die ECID dauerhaft, nachdem die Mobile-App installiert wurde, und Sie müssen die App neu installieren, um eine neue ECID zu generieren. Im Web wird nach dem Löschen des Browser-Cookies eine neue ECID generiert. |
| Begrenzung des täglichen Hochladevorgangs | Adobe empfiehlt, Kundenattribute nur einmal pro Tag zu aktualisieren. Sie müssen mindestens 24 Stunden warten, um eine weitere Kundendatendatei für denselben Satz von Profilen hochzuladen. |
| Benutzerspezifische Analytics-ID (`s.visitorID`) | In Analytics können Besucher identifiziert werden, indem eine Kunden-ID mit `s.visitorID` festgelegt wird. Integrationen, in denen Analytics-Daten mit dem ID-Dienst exportiert oder importiert werden, funktionieren jedoch nicht, wenn ein Besucher anhand `s.visitorID.`<br>dieser Funktion identifiziert wird. Dazu gehören unter anderem freigegebene Audiencen, Analytics für Adobe Zielgruppe (A4T) und Kundenattribute.<br>Bei diesen Integrationen wird die Festlegung einer benutzerdefinierten Analytics ID nicht unterstützt. |
| Zeichenlängenbeschränkungen in Analytics | Beim Erstellen eines Analytics-Abonnements werden die Feldlängen für die hochgeladenen Dateien auf 255 gekürzt. |

## Häufig gestellte Fragen zu Kundenattributen  {#section_E47866EEA83348E09FE43CEC5E44C461}

| Frage | Antwort |
|--- |--- |
| Kann ich Benachrichtigungen zum Upload-Status für Kundenattribute erhalten? | Ja. Weitere Informationen finden Sie unter [Benachrichtigungen verwalten](https://docs.adobe.com/content/help/en/core-services/interface/manage-users-and-products/organizations.html#concept_0105453AD71847B8BFCAF4A40915F157). |
| Was muss ich tun, um mit Kundenattributen zu beginnen? | <ol><li>Lassen Sie sich freischalten. Wenn Sie Analytics-Kunde sind, stellt Adobe Sie für Kundenattribute bereit. Wenn Sie nur Adobe-Zielgruppe verwenden und nicht über Analytics verfügen, müssen Sie die Bereitstellung der Hauptdienste anfordern, indem Sie sich an die Kundenunterstützung wenden.</li> <li>Unterhalten Sie sich mit Ihrem CRM-Team. Finden Sie heraus, welche Art von Kundendaten zur Verfügung stehen, die für die Verwendung in Analytics und der gesamten Experience Cloud interessant sind.</li><li>Implementieren Sie Hauptdienste. Anweisungen zur Modernisierung Ihrer Implementierung finden Sie unter [Aktivieren von Lösungen für Hauptdienste](https://docs.adobe.com/content/help/en/core-services/interface/about-core-services/core-services.html) . (Wichtige Informationen finden Sie im Abschnitt zum Synchronisieren von Kunden-IDs.)</li></ol> **Hinweis:** Eine FAQ-Rubrik für Administratoren zur Implementierung von Hauptdiensten finden Sie  [hier](https://docs.adobe.com/content/help/en/core-services/interface/manage-users-and-products/faq.html#concept_13219B4E51784577B6FF78AAA203DE91). |
| Wie viele Kundenattribute darf ich verwenden? | You can upload hundreds of `.csv` columns to the customer attribute service. Bei der Konfiguration von Abonnements und der Auswahl von Attributen gelten jedoch die folgenden Einschränkungen (pro Report Suite), je nach den Lösungen, die Sie besitzen:  <ul><li>Foundation: 0</li><li>Select: 3</li><li>Prime: 15</li><li>Ultimate: 200</li><li>Standard: 3 insgesamt</li><li>Premium: 200</li><li>Adobe Zielgruppe Standard: 5</li><li>Adobe Zielgruppe Premium: 200</li></ul> |
| Ist eine Migration zum Experience Cloud ID-Dienst erforderlich? | Die Migration hängt von den verwendeten Lösungen ab. <ul><li>Adobe Analytics:  Streng empfohlen </li><li>Adobe-Zielgruppe: Erforderlich. </li></ul><br> Die Verwendung des ID-Diensts verbessert die Funktionalität und eröffnet die Möglichkeit, die neuesten Experience Cloud-Funktionen zu nutzen, einschließlich Echtzeit-Audiencen, der Modernisierung der Adobe-Zielgruppe, der Analytics-Integration und der Video Heartbeat-Verfolgung. <br> Weitere Informationen finden Sie unter [Aktivieren Ihrer Lösungen für Hauptdienste](https://docs.adobe.com/content/help/en/core-services/interface/about-core-services/core-services.html). <br>** Hinweis:** Der [Experience Cloud ID-Dienst](https://docs.adobe.com/content/help/en/id-service/using/intro/overview.html) ist die modernisierte Implementierung des zuvor als _Analytics-Besucher-ID-Dienst bekannten Dienstes._ |
| In welcher Beziehung steht die Kundenattributfunktion zu Adobe Audience Manager? | Audience Manager kann zwar Daten zur Identifizierung von Audiencen empfangen, kann jedoch keine Analysefunktionen durchführen, die Attribute mit historischen Verhaltensdaten verknüpfen oder die in Adobe Analytics verfügbaren Berichte-, Analyse- und Segmentierungsfunktionen bereitstellen. Der Hauptdienst &quot;Personen&quot;ermöglicht die Verknüpfung von Rich-Daten aus verschiedenen Lösungen und die Verknüpfung mit einer einzigen ID zur Verwendung in der Experience Cloud. <br>In Adobe Zielgruppe erscheinen Kundenattribute als individuelle Attribute, die mit anderen Regeln kombiniert werden können, um Audiencen zu erstellen. Im Hauptdienst &quot;Personen&quot;freigegebene Audiencen sind vollständige Audiencen, die nicht geändert werden können. |
| **(Nur Analytics)** Worin unterscheidet sich diese Funktion von der in Analytics Premium bereitgestellten Funktionalität? | In der Vergangenheit haben Kunden, die Kundenattributdaten mit Analytics-Daten kombinieren möchten, sich bei dieser Funktion stark auf das Data Workbench-Tool verlassen. Kundenattribute stellen diese Funktion einer breiteren Audience zur Verfügung, indem sie Kundenattribute als Dimensionen und Metriken in Reports &amp; Analysen, Ad-hoc-Analysen und ReportBuilder bereitstellen. Analytics Standard-Kunden haben Zugriff auf Kundenattribute, jedoch mit eingeschränkten Funktionen. Die volle Funktionalität steht Analytics Premium-Kunden zur Verfügung. |
| **(Nur Adobe-Zielgruppe)** Kann ich Daten für Kunden, die Adobe Zielgruppe noch nie gesehen hat, vorab laden oder hochladen? | Ja. Wenn der Besucher die erste Anforderung an die Adobe-Zielgruppe stellt, ruft das System die vorhandenen Informationen zu ihnen aus den Kundenattributen ab und verwendet diese Daten für das Targeting. **Hinweis:**  Das Abrufen dieser Daten kann bis zu 20 Minuten nach der ersten Interaktion des Besuchers mit der Adobe-Zielgruppe dauern. |
| **(Nur Adobe-Zielgruppe)** Kann ich eine Super-Audience erstellen, indem ich Kundenattributdaten mit Daten zur freigegebenen Audience kombiniere? | Nein. Freigegebene Audiencen sind eine abgeschlossene Audience. |
| **(Nur Adobe-Zielgruppe)** Wie unterscheidet sich die Funktionalität der Kundenattribute von der Bulk Profil API der Adobe Zielgruppe? | Mit der Massen-Profil-API können Adobe Zielgruppe-Profil direkt über die API aktualisiert werden, entweder für ein einzelnes Profil oder stapelweise. Die Funktion ähnelt den Kundenattributen mit folgenden Hauptunterschieden:<ul><li>Die Profil-API ist ein REST-API-Aufruf und Kundenattribute verwenden FTP.</li><li>Die Profil-API von Adobe Zielgruppe sendet nur Daten an die Adobe-Zielgruppe und nicht an die gesamte Experience Cloud.</li><li>Kundenattribute bieten eine einfache Oberfläche zum Erstellen und Verwalten dieser externen Daten.</li></ul> |
| **(Nur Adobe-Zielgruppe)** Verlängert das Hochladen von Daten aus Kundenattributen in die Adobe-Zielgruppe die Laufzeit des Profils des Adobe Zielgruppe-Besuchers? | Ja. Weitere Informationen finden Sie unter [Lebensdauer von Besucherprofilen](https://docs.adobe.com/content/help/en/target/using/audiences/visitor-profiles/visitor-profile.html) in der Hilfe zu Adobe Target. |
| **(Nur Adobe-Zielgruppe)** Kann ich Zielgruppen zu den in Kundenattributen hochgeladenen Daten vornehmen, sobald der Besucher durch die Kunden-ID identifiziert wurde? | Ja. Beim Server-Aufruf an Adobe Zielgruppe, der die Mbox-Drittanbieter-ID enthält, stehen alle Kundenattributdaten zur Verfügung. |
| **(Nur Adobe-Zielgruppe)** Was bedeutet die Spalte &quot; **[!UICONTROL Synchronisierungsstatus]** &quot;für in der Kundenattributquelle hochgeladene Dateien? | Die Anzahl der von Adobe Zielgruppe veröffentlichten und synchronisierten Datensätze kann durch Klicken auf das Symbol &quot;Synchronisierungsstatus&quot;für eine bestimmte Attributdatei angezeigt werden. `Sync %` ist eine Echtzeitmetrik, die den Prozentsatz der Profil angibt, die in Adobe Zielgruppe synchronisiert wurden.<br> **Hinweis:** Die Synchronisierung von Attributen mit Adobe Zielgruppe kann bis zu 24 Stunden dauern. |
| Was stellen die Metriken zum Hochladen von Dateien in der Quelle der Kundenattribute dar? | Sie können den Status von Attributen, die in Kundenattribute hochgeladen wurden, mithilfe der folgenden Metriken überprüfen: <ul><li>Aufzeichnungen:  Anzahl der Datensätze in der Attributdatei.</li><li>**Neue Datensätze:** Anzahl der neuen Datensätze, die in der Attributdatei vorhanden sind.</li> <li>**Aktualisierte Datensätze:** Anzahl der Datensätze, die bereits in Kundenattributen mit aktualisierten Werten in der Datei vorhanden sind.</li><li>**Alle Daten (Datensätze):** Gesamtzahl der erfolgreich in Kundenattribute hochgeladenen Datensätze.</li></ul> |
